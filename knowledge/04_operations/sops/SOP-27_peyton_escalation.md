# SOP-27 — Peyton Escalation Triggers

**Owner:** Service Manager (Ryan)  
**Resource:** Peyton (part-time diagnostics)  
**Last verified:** 2026-07-12  
**Status:** Draft — ready for review

## Purpose

Use Peyton when needed — not as default WIP — with a timeboxed diagnostic block.

## Triggers (any one)

- Intermittent driveability after standard diag (SOP-22)
- Controller/monitor programming beyond handheld reset
- Repeated comeback on same electrical complaint
- Fleet/HOA multi-cart pattern

## Steps

1. Primary tech completes standard path notes.
2. Ryan schedules Peyton block; card row **P** on board.
3. One-line symptom on card/HCP.
4. Timebox the block; findings → estimate handoff (SOP-24).
5. Do not leave Peyton jobs as endless unassigned WIP.

## Done when

Findings documented; next lane clear; Peyton block closed.
